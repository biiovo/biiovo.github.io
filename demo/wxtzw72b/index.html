Docker镜像、容器、网络及数据管理的核心操作

一、基础管理命令

  1. 查看Docker版本
    docker --version
    输出示例：Docker version 20.10.21, build baeda1f
  2. 显示系统信息
    docker info
    显示镜像数、容器数、存储驱动等详细信息
  3. 获取命令帮助
    docker help [command]
    示例：docker help run

二、镜像管理命令

  1. 拉取镜像
    docker pull [镜像名]:[标签]
    示例：docker pull nginx:latest
  2. 列出本地镜像
    docker images [-a] [-q]
    -a：显示所有镜像（含中间层）；-q：仅显示ID
  3. 删除镜像
    docker rmi [镜像ID或名] [-f]
    -f：强制删除（即使被容器使用）
  4. 构建镜像
    docker build -t [镜像名]:[标签] [Dockerfile路径]
    示例：docker build -t myapp:1.0 .
  5. 推送镜像到仓库
    docker push [镜像名]:[标签]
    需提前登录仓库（docker login）

三、容器管理命令

  1. 创建并启动容器
    docker run [选项] [镜像名] [命令]
    常用选项：
    • -d：后台运行
    • -p 主机端口:容器端口：端口映射
    • --name 容器名：自定义名称
    示例：docker run -d -p 8080:80 --name mynginx nginx
  2. 查看容器状态
    docker ps [-a] [-q]
    -a：显示所有容器（含停止的）
  3. 启动/停止/重启容器
    docker start/stop/restart [容器名或ID]
    示例：docker restart mycontainer
  4. 进入容器
    docker exec -it [容器名或ID] /bin/bash
    以交互模式进入容器
  5. 删除容器
    docker rm [容器名或ID] [-f]
    -f：强制删除运行中的容器

四、数据卷与网络管理

  1. 创建数据卷
    docker volume create [卷名]
    示例：docker volume create mydata
  2. 挂载数据卷
    docker run -v [主机路径]:[容器路径] [镜像名]
    示例：docker run -v /host/data:/container/data nginx
  3. 创建自定义网络
    docker network create [网络名]
    示例：docker network create mynet
  4. 连接容器到网络
    docker network connect [网络名] [容器名]
    示例：docker network connect mynet mycontainer

五、清理与优化命令

  1. 清理未使用资源
    docker system prune [-a] [-f]
    -a：清理未使用的镜像、容器、网络和卷；-f：强制清理
  2. 导出/导入容器
    docker export [容器ID] > [文件名].tar
docker import [文件名].tar [镜像名]:[标签]
    示例：导出容器为tar文件

六、Docker Compose命令

  1. 启动服务
    docker-compose up [-d]
    -d：后台运行
  2. 停止并删除服务
    docker-compose down
    示例：docker-compose down
  3. 构建镜像
    docker-compose build
    示例：docker-compose build
